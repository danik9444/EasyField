# Security policy

## Supported versions

The current `1.2.x` line receives security fixes. Unsupported development
snapshots and older versions may be asked to upgrade before a report is
investigated.

## Reporting a vulnerability

Please use the repository's **Security → Report a vulnerability** flow to open
a private GitHub Security Advisory. Do not post exploit details, credentials,
private media, or an unpatched vulnerability in a public issue.

Include the affected version, macOS and Resolve versions, reproduction steps,
impact, and any safe proof of concept. Reports are reviewed as capacity allows;
no fixed response or remediation SLA is promised.
